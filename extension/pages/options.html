Add-ons

Edit this code
  • "autoFoldLevels" - How many levels to automatically fold. Default is 0.
  • "prependHeader" - Comment header with URL and timestamp (boolean). Default true
  • "maxJsonSize" - value in kbytes (integer). Default 400
  • "alwaysFold" - Fold from the second level. A button to unfold all will appear if the value is true (boolean). Default false
  • "prefoldKeys" - List of keys to prefold. Type is {"key": <keyToFold>, "autoFoldUntil": <count>} Level count is the number of levels to automatically fold.
  • "alwaysRenderAllContent" - Disables the custom search and renders all content. Be aware that with this option large files will have a significant performance reduction (boolean). Default false
  • "sortKeys" - Sort objects by key (boolean). Default false
  • "clickableUrls" - Make URLs clickable (boolean). Default true
  • "wrapLinkWithAnchorTag" - Add URLs as anchor tag (boolean). Default false
  • "openLinksInNewWindow" - Open URLs in the same window/tab (boolean). Default true
  • "autoHighlight" - Highlights the JSON from the beginning. When set to false it still possible to highlight the JSON from the raw page (boolean). Default true
  • "parseJsonStrings" - Finds string values that are JSON strings and parses them into the object as well.
  • "parseJsonStringDepth" - When parsing embedded JSON string values, this determines how deep to go. Larger values require additional traversals of the data. Once no JSON strings are found, parsing stops.

This is a json object defining how to summarize folded JSON trees 
                [ // each item is checked for a match against the current json object (in order of definitions below). All of these are optional
                  {
                    keyToMatch: ...                        // object[key] must exist else continue
                    valueToMatch: ...                      // object[key] must be valueToMatch else continue
                    label: "..."                           // append label to the summary
                    keyToRecurse: "..."                    // recursively evaluate object[keyToRecurse]
                    recurseDefinitions: [...]              // use this definition as the fold summarizer for this recurse
                    keysToExtract: [key1, ...]             // extract the keys for object[key1], object[key2], ...
                    extractValues: true|false              // if true, then extract the values for object[key], ...
                    extractLabel: ""                       // label to prepend before extracted keys/values
                    continueAfterMatch: true|false         // if true, continue evaluating for more matches
                    dontMatchWhenSummarizing: true|false   // if true, don't evaluate this rule if we are summarizing a folded array
                  }
                ]
